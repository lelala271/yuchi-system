# 常见问题排查

本文档解决的问题是：当前端能打开但实时链路不通、控制无反馈、登录失败或者跨域报错时，应该按什么顺序检查。

一句话结论：先分清是哪条链路出问题，再按“现象、原因、检查方法、解决办法”排，不要把 HTTP、WebSocket 和 MQTT 混着猜。

## 1. 总排查顺序

1. 先看后端是否正常启动。
2. 再看前端是否成功登录。
3. 再分开检查 HTTP、WebSocket、MQTT 三条链路。
4. 最后再检查模拟器、设备主题和反向代理。

## 2. 常见问题总表

| 现象 | 可能原因 | 检查方法 | 解决办法 |
| --- | --- | --- | --- |
| 页面能打开但无法登录 | 用户名密码错误、登录限流、后端未启动 | 看 `/api/auth/login` 响应码和后端日志 | 检查初始密码，等待限流窗口结束，确认后端正常运行 |
| 登录成功但刷新后掉线 | token 失效或后端密钥变化 | 检查 `/api/auth/me`、浏览器存储和后端密钥文件 | 重新登录，确认 `AUTH_TOKEN_SECRET` 未意外变化 |
| WebSocket 一直断开 | token 无效、Origin 不允许、地址配置错 | 看浏览器 Console、Network 握手和后端日志 | 检查 `VITE_WS_URL`、跨域来源、登录状态 |
| MQTT 页面状态未连接 | MQTT WebSocket 地址错误、鉴权失败、端口未开放 | 看前端日志和 Broker 输出 | 检查 `VITE_MQTT_WS_URL`、JWT、`8888` 端口 |
| 页面有车辆但图表不更新 | 指标未写入、MQTT 主题未到达、模拟器关闭 | 查 `/api/metrics/latest`、Broker 日志 | 检查模拟器、设备上报和主题 |
| 控制命令成功返回但设备没动作 | 设备未订阅控制主题或设备凭据错误 | 看 MQTT 订阅和 Broker 日志 | 检查设备是否订阅 `yuchi/vehicle/+/control` |
| 真实设备连不上 MQTT | 用户名密码不对、匿名被禁、主题超白名单 | 看 Broker 鉴权和授权日志 | 使用设备专用身份，检查主题命名 |
| 跨域请求被拒绝 | `CORS_ORIGINS` 未配置或域名不匹配 | 看响应是否 `403` | 把真实前端域名加入白名单 |
| 生产环境页面正常但实时不通 | 反向代理未转发升级协议 | 检查网关配置和握手请求 | 开启 WebSocket `Upgrade` / `Connection` 转发 |

## 3. 按链路排查

### 3.1 HTTP 链路

先检查：

- `/api/health`
- `/api/auth/login`
- `/api/auth/me`

如果这三项都不通，先不要去看 WebSocket 或 MQTT。

### 3.2 WebSocket 链路

重点检查：

- 地址是否正确
- 是否带了 `auth.<token>`
- 来源域名是否被允许
- 反代是否支持升级协议

### 3.3 MQTT 链路

重点检查：

- 页面还是设备在连
- 页面是否用了用户名 + JWT
- 设备是否用了设备专用身份
- 主题是否在白名单里
- 端口是否被占用或未暴露

## 4. 和模拟器有关的问题

| 现象 | 原因 | 处理办法 |
| --- | --- | --- |
| 本地演示完全没数据 | 模拟器关闭 | 检查 `ENABLE_SIMULATOR` |
| 生产环境仍有虚拟车辆 | 模拟器未关闭 | 生产设为 `ENABLE_SIMULATOR=false` |
| 真实设备和模拟器数据混在一起 | 演示和生产未隔离 | 关闭模拟器或分环境部署 |

## 5. 和账号有关的问题

| 现象 | 原因 | 处理办法 |
| --- | --- | --- |
| 初始管理员密码找不到 | 没看运行时密钥文件 | 查看 `backend/data/runtime-secrets.json` |
| 旧管理员密码失效 | 系统自动轮换历史弱口令 | 使用新的运行时密码登录 |
| 无法注册新用户 | 注册默认关闭 | 明确设置 `ALLOW_REGISTRATION=true` 后再试 |

## 6. 最终排错清单

1. 后端有没有启动成功。
2. 健康检查是否正常。
3. 登录是否成功。
4. WebSocket 是否已连接。
5. MQTT 页面连接是否已连接。
6. 模拟器或设备是否真的在发数据。
7. 设备主题是否符合白名单。
8. 反向代理是否转发了升级协议。

## 7. 最终记忆表

| 问题类型 | 先看什么 |
| --- | --- |
| 登录问题 | `/api/auth/login` |
| 页面无实时数据 | WebSocket 连接状态 |
| 设备无实时数据 | MQTT 连接和主题 |
| 本地无演示数据 | 模拟器 |
| 线上实时断流 | 反向代理和跨域 |
